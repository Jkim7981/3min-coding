# AI 문제 품질 평가 보고서

## 목적
- 생성된 AI 문제의 난이도 적절성, 문장 자연스러움, 힌트 유용성, 해설 명확성을 점검한다.
- 실제 수업 자료 기반 문제 생성 결과를 C 담당 관점에서 기록해 Day 4 체크포인트에 반영한다.

## 테스트 범위
- 대상 과목: 파이썬 기초, 자바스크립트 기초
- 표본 수: 최소 10문제 이상
- 문제 유형: 개념 문제, 코딩 문제
- 확인 환경: 로컬 개발 서버, 배포 URL

## 평가 기준
| 항목 | 확인 내용 | 합격 기준 |
| --- | --- | --- |
| 난이도 적절성 | easy, medium, hard가 체감상 구분되는가 | 10문제 중 8문제 이상 일치 |
| 오타/문법 오류 | 지문, 힌트, 해설의 문법과 띄어쓰기 | 심각한 오류 0건 |
| 힌트 유용성 | 정답을 직접 노출하지 않고 방향을 주는가 | 과도한 정답 노출 0건 |
| 해설 명확성 | 왜 틀렸는지와 왜 정답인지 설명하는가 | 이해 가능 수준 80% 이상 |
| UI 적합성 | 카드, 버튼, 모달 안에서 문장이 과도하게 길지 않은가 | 줄바꿈 붕괴 0건 |

## 테스트 기록 템플릿
| 번호 | 과목 | 유형 | 난이도 | 결과 | 메모 |
| --- | --- | --- | --- | --- | --- |
| 1 | 자바스크립트 기초 | 개념 | medium | 통과 | 이벤트 리스너 설명이 자연스러움 |
| 2 | 자바스크립트 기초 | 코딩 | medium | 보완 필요 | 정답 형식 예시가 부족함 |
| 3 | 파이썬 기초 | 개념 | easy | 통과 | 보기 문장 길이 적절 |

## 주요 발견 사항
- 개념 문제는 설명형 문장보다 비교형 보기에서 가독성이 좋다.
- 코딩 문제는 빈칸 정답 형식이 짧을수록 사용자가 이해하기 쉽다.
- 해설은 2~3문장 이내로 유지할 때 UI 밀도가 안정적이다.
- 힌트 버튼은 강한 대비가 있어야 실제 사용 시 잘 눌린다.

## 이슈 리스트
- 일부 난이도 라벨이 기대보다 높거나 낮게 느껴질 수 있어 조정 여지가 있다.
- 긴 해설은 모바일 화면에서 카드 높이를 크게 늘릴 수 있다.
- 실제 수업 자료 기반 문제는 전문 용어 표기를 한번 더 검수하는 편이 안전하다.

## 후속 제안
- B 담당과 함께 few-shot 예시를 과목별로 나눠 품질을 비교한다.
- Day 5 전 최종 배포본에서 모바일 기준 10문제 샘플을 다시 확인한다.
