# AI 빌딩 리포트 초안

## 1. 프로젝트 개요
3분 코딩은 짧은 학습 단위 안에서 개념 문제와 코딩 문제를 반복적으로 풀며 학습할 수 있게 돕는 웹 앱이다. 학생은 과목과 회차를 따라 문제를 풀고, 시스템은 정답 여부와 힌트, 해설을 통해 복습 흐름을 제공한다.

## 2. 해결하고자 한 문제
- 한 번에 긴 학습을 하기 어려운 사용자도 짧은 세션으로 꾸준히 학습할 수 있어야 한다.
- 오답 이후 바로 학습 맥락을 유지한 채 힌트와 해설을 받을 수 있어야 한다.
- 난이도와 문제 유형이 섞여도 화면이 복잡하지 않게 보여야 한다.

## 3. AI 활용 전략
- 문제 생성: 수업 자료를 기반으로 개념 문제와 코딩 문제를 생성한다.
- 해설 생성: 오답 시 왜 틀렸는지, 정답이 왜 맞는지 짧고 쉬운 한국어로 설명한다.
- 난이도 적응: 최근 풀이 결과를 기반으로 easy, medium, hard를 조절한다.

## 4. UI/테스트 관점에서의 역할
- 로딩 화면, 홈 화면, 개념/코딩 문제 화면, 힌트 팝업, 결과 화면 퍼블리싱
- 더미 데이터 설계 및 화면 연결
- 실제 생성 문제의 품질 검토와 이슈 정리
- 제출 직전 UI 일관성, 가독성, 버튼 대비 점검

## 5. 기술 스택
- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase
- OpenAI API
- Vercel

## 6. 구현 포인트
- 홈 화면에서 사용자 정보 카드, 주간 정답률, 수강 과목 카드로 학습 상태를 바로 파악할 수 있게 구성했다.
- 문제 화면은 개념형과 코딩형을 같은 레이아웃 안에서 다르게 보여 주되, 버튼 위치와 간격은 통일했다.
- 결과 화면은 정답 여부, AI 생성 해설, 다음 이동 버튼을 한 묶음으로 배치해 흐름이 끊기지 않게 했다.

## 7. 품질 점검 메모
- 버튼은 모바일에서도 눌리기 쉽게 충분한 높이와 대비를 유지했다.
- 차트와 카드 간격은 8px, 16px, 24px 단위로 정리해 일관성을 맞췄다.
- 긴 문장은 카드 폭 안에서 자연스럽게 줄바꿈되는지 확인했다.

## 8. 향후 개선 방안
- 실제 API 연결 이후 통계 차트를 실데이터 기반으로 전환
- 문제 품질 평가표를 자동화된 체크리스트로 확장
- 접근성 검토를 통해 키보드 이동성과 ARIA 보강

## 9. 첨부 예정 자료
- 홈 화면 스크린샷
- 문제 풀이 화면 스크린샷
- 결과 및 해설 화면 스크린샷
- AI 문제 품질 평가 표
