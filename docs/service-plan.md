## 프로젝트 개요

### 💡 프로젝트 이름
**3분코딩 (Three-Min-Coding)**

### 📖 프로젝트 설명
AI 기반 학습 지원 웹앱으로, "학원생들이 매일 단 3분 만에 수업 내용을 복습할 수 있도록 돕습니다." 강사가 업로드한 수업 자료(텍스트, 코드 파일, 수업 영상)를 AI가 분석하여 맞춤형 문제를 자동 생성하고 학생들의 스마트폰으로 매일 배포합니다.

### ✨ 핵심 가치
- **⏱️ 부담 없는 학습**: 하루 3분으로 부담 없이 꾸준한 복습 가능
- **🎯 맞춤형 문제**: 내가 듣는 수업 내용에 딱 맞는 AI 생성 문제
- **🤖 자동화된 운영**: 강사는 자료만 올리면 AI가 자동으로 문제 생성 및 배포
- **📈 학습 효과 극대화**: SM-2 간격 반복 알고리즘으로 장기 기억 형성

### 👥 타겟 사용자

| 구분 | 대상 |
|------|------|
| **주 사용자** | 코딩 학원 수강생 (복습이 필요하지만 시간이 부족한 학생) |
| **부 사용자** | 학원 강사 및 운영자 (효율적인 복습 시스템 도입을 원하는 교육기관) |

## 기능 목록

### 👨‍🎓 학생(수강생)용 기능

#### 1️⃣ 과목별 AI 맞춤형 문제 제공
- 학원 DB 연동으로 수강 중인 과목만 자동 표시
- 과목 탭 선택 후 해당 과목의 AI 생성 문제 제공
- **2가지 문제 유형**:
  - 개념 문제: 이론 지식, 용어 이해 테스트 (객관식/단답형)
  - 코딩 문제: 빈칸 채우기 형식 + Judge0 CE로 실제 코드 실행
- 과목별 학습 진도 시각화

#### 2️⃣ 2단계 오답 플로우 🎯

| 단계 | 내용 |
|------|------|
| 1차 정답 | 칭찬 메시지 + 다음 문제 |
| 1차 오답 | "다시 한번 생각해봐!" 메시지 + 힌트 보기 버튼 + 재시도 입력창 |
| 2차 정답 | 정답 처리 + 해설 |
| 2차 오답 | AI 자동 생성 해설 (왜 틀렸는지, 왜 이게 정답인지, 관련 개념 설명) |

**힌트 시스템**:
- 1차 오답 시에만 힌트 버튼 노출
- 개념 문제: 정답 키워드 첫 글자 + 개념 설명
- 코딩 문제: 빈칸 단어 첫 글자 + 어떤 개념을 사용해야 하는지 설명

#### 3️⃣ 실시간 코드 실행 (Judge0 CE) 💻
- 지원 언어: Python, JavaScript, Java
- CE(Community Edition) 무료 API 사용 (https://ce.judge0.com)
- 학생이 빈칸을 채우면 실제 코드를 실행하여 결과 표시
- 실행 타임아웃: 15초

#### 4️⃣ 적응형 난이도 시스템 📊

| 최근 10문제 정답률 | 난이도 |
|------------------|--------|
| ≥ 80% | 어려운 문제 (hard) |
| 50~79% | 중간 난이도 (medium) |
| < 50% | 쉬운 문제 (easy) |

- 과목별로 독립적으로 계산
- 첫 시작 기본값: medium

#### 5️⃣ 스마트 푸시 알림 (PWA)
- 매일 오전 9시(KST) 알림 자동 전송 (Vercel Cron)
- 사용자 맞춤 시간 설정 가능
- VAPID 기반 Web Push 구현

#### 6️⃣ SM-2 간격 반복 복습 시스템
- 오답 문제를 SM-2 알고리즘으로 복습 스케줄링
- quality 점수: 1차 정답(5), 힌트 후 정답(3), 2차 오답(1)
- 복습 간격: 1일 → 6일 → interval × EF (최대 60일)
- 대시보드에서 오늘 복습할 문제 표시

#### 7️⃣ 주간/월간 취약점 분석 리포트 📈
- 오답 패턴 분석 (OpenAI GPT-4o-mini)
- 개선을 위한 구체적 조언
- 취약 개념 Top N 표시

#### 8️⃣ 학습 현황 대시보드
- 일일/주간 학습 통계
- 과목별 정답률 및 취약 개념 분석
- 연속 학습 일수(스트릭) 표시
- 오늘의 문제 (신규 3개 + SM-2 복습 2개)

#### 9️⃣ 공지사항 — 자격증 시험 일정 📅
- 정보처리기사, 빅데이터분석기사, SQLD, ADsP, 리눅스마스터 일정 제공
- 달력 뷰 + 목록 뷰 전환 가능
- 시험 접수·시험·합격발표 유형별 색상 구분
- data.go.kr 한국산업인력공단 API 연동 (24시간 캐시)

#### 🔟 모바일 최적화 (PWA)
- PWA 기반 앱 설치 (홈 화면 아이콘)
- 전체 화면 실행
- 어느 화면에서나 우측 상단 햄버거 메뉴(☰)로 즉시 탭 이동

### 👨‍🏫 학원(관리자)용 기능

#### 1️⃣ 수업 자료 업로드 시스템
- **텍스트/코드 파일** (.txt, .md, .ipynb) 업로드
- **수업 영상 파일** (.mp4, .mov, .avi, .mkv, .webm) 업로드

- 드래그 앤 드롭 방식 지원

- 적응형 난이도 자동 조절
- 힌트·해설·개념 태그 자동 생성
- 생성된 문제 미리보기 및 확인 기능

#### 4️⃣ 과목 및 수강생 관리
- 과목별 수강생 현황 확인
- 수업 회차 자료 이력 관리
- 학원코드 기반 회원가입으로 수강 과목 자동 연결

## 웹앱 작동 원리

### 🌐 웹앱이란?
웹 브라우저에서 실행되는 앱입니다. 앱스토어 다운로드 없이 URL을 통해 접속하면 바로 사용할 수 있습니다.

**장점**:
- ✅ 설치 불필요
- ✅ 자동 업데이트
- ✅ 크로스 플랫폼 (iOS, Android, PC)
- ✅ 빠른 개발

### 📱 PWA (Progressive Web App)란?
웹앱을 스마트폰 홈 화면에 '앱처럼 설치'할 수 있게 해주는 기술입니다.

**PWA로 할 수 있는 것**:
- 🏠 홈 화면 아이콘 추가
- 📺 전체 화면 실행
- 🔔 푸시 알림
- ✈️ 오프라인 지원

### 🔄 3분코딩 작동 흐름

**Step 1. 강사가 수업 자료 업로드**
- 텍스트/코드 파일 또는 수업 영상 파일 업로드

**Step 2. 영상이라면 — 키프레임 자동 분석**
- 브라우저에서 Canvas API로 화면이 바뀌는 순간의 프레임만 추출
- GPT-4o Vision이 각 프레임의 코드·슬라이드 내용을 텍스트로 변환

**Step 3. AI가 수업 자료 분석 후 복습 문제 자동 생성**
- OpenAI GPT-4o가 내용을 분석하여 개념 문제 + 코딩 문제 생성
- 힌트, 해설, 개념 태그 자동 포함

**Step 4. 학생 스마트폰으로 문제 배포**
- 생성된 문제가 해당 과목 수강생에게만 전송
- 매일 오전 9시(KST) Vercel Cron + Web Push로 알림 발송

**Step 5. 틀린 문제는 SM-2 알고리즘으로 재출제**
- 오답률과 시도 횟수에 따라 복습 간격 자동 조절
- 오늘 복습할 문제를 대시보드에서 바로 확인

### 🏗️ 기술 구조

## AI 문제 생성 테스트 계획

### 🎯 테스트 목적
실제 학원 강사님의 수업 자료(텍스트, 코드 파일, 수업 영상)를 사용해 AI가 '실전에 쓸 만한' 문제를 만들어내는지 검증하고 프롬프트를 개선합니다.

### 📋 테스트 진행 방식

#### 1단계: 수업 자료 수집 (Day 1~2)
- 실제 학원 강사님께 협조 요청
- 다양한 난이도와 주제의 자료 확보 (최소 5개 이상)
- 텍스트 자료 + 수업 영상 자료 혼합 테스트

#### 2단계: AI 문제 생성 (Day 3~4)
- 수집한 자료를 시스템에 업로드
- 영상의 경우 키프레임 분석 → 텍스트 변환 → 문제 생성 전체 파이프라인 테스트
- 각 자료당 3~5개 문제 생성
- 개념 문제(객관식/단답형), 코딩 문제(빈칸 채우기) 골고루 테스트

#### 3단계: 품질 평가 (Day 5)

| 평가 항목 | 기준 | 점수 |
|---------|------|------|
| **정확성** | 수업 내용과 일치, 오답/오류 없음 | /5 |
| **난이도** | 복습용으로 적절, 너무 쉽거나 어렵지 않음 | /5 |
| **실전성** | 단순 암기 아닌 응용 능력 테스트 | /5 |
| **명확성** | 문제 지문이 명확하고 이해하기 쉬움 | /5 |

#### 4단계: 개선 및 최적화
- 평가 결과를 바탕으로 AI 프롬프트 조정
- 영상 키프레임 추출 임계값 최적화
- 강사 피드백 반영

### 📊 예상 결과
